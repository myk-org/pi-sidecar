Tracking Token Usage and Costs

Track how many tokens your AI calls consume and what they cost by registering a usage recorder callback — useful for billing dashboards, cost alerts, analytics pipelines, or debugging slow prompts.

Prerequisites

  • A running pi-sidecar server (see Getting Started with pi-sidecar)
  • The Python client library installed (pip install pi-sidecar-client)

Quick Example

import asyncio
from pi_sidecar_client import AIResult, call_ai_once, set_usage_recorder


async def log_usage(*, request_id, result, call_type, prompt_chars, ai_provider, ai_model):
    """Print token counts after every AI call."""
    if result.usage:
        print(f"[{request_id}] {result.usage.input_tokens}in / {result.usage.output_tokens}out"
              f" — ${result.usage.cost_usd or 0:.6f}")


async def main():
    set_usage_recorder(log_usage)

    result = await call_ai_once(
        "What is the speed of light?",
        ai_provider="gemini",
        ai_model="gemini-2.5-flash",
    )
    await result.record_usage(
        request_id="req-001",
        call_type="question",
        ai_provider="gemini",
        ai_model="gemini-2.5-flash",
    )
    print(result.text)


asyncio.run(main())

How It Works

Usage tracking has two parts:

  1. Register a callback — call set_usage_recorder() once at startup.
  2. Record after each call — call result.record_usage() after every AI call to send data to your callback.

Recording is explicit and opt-in. If you forget to call record_usage(), no data is captured. This gives you full control over when and with what metadata usage is recorded.

Step-by-Step Setup

1. Define Your Recorder Callback

Your callback receives keyword arguments describing the completed AI call. It can be either sync or async:

async def my_recorder(
    *,
    request_id: str,      # Your identifier for this request
    result: AIResult,      # The full AI result (text, usage, success, error)
    call_type: str,        # Your label (e.g. "summarize", "translate", "review")
    prompt_chars: int,     # Character count of the prompt sent
    ai_provider: str,      # Provider used (e.g. "gemini", "cursor", "claude")
    ai_model: str,         # Model used (e.g. "gemini-2.5-flash")
) -> None:
    ...

All parameters are passed as keyword arguments. Your callback must accept exactly these six parameters.

2. Register It at Startup

Call set_usage_recorder() once, before making any AI calls:

from pi_sidecar_client import set_usage_recorder

set_usage_recorder(my_recorder)

You can replace the recorder at any time by calling set_usage_recorder() again. Only one callback is active at a time.

3. Record Usage After Each Call

After every call_ai_once() or call_ai() result, call record_usage() with your metadata:

result = await call_ai_once("Summarize this document", ai_provider="gemini", ai_model="gemini-2.5-flash")

await result.record_usage(
    request_id="job-42",
    call_type="summarize",
    ai_provider="gemini",
    ai_model="gemini-2.5-flash",
)

Tip: prompt_chars defaults to 0 if omitted. Pass prompt_chars=len(prompt) to track input size for your own analytics.

Reading the Usage Data

Every AIResult includes a usage field with token counts, cost, and timing:

Field Type Description
input_tokens int Tokens sent to the model
output_tokens int Tokens generated by the model
cache_read_tokens int Tokens served from cache
cache_write_tokens int Tokens written to cache
cost_usd float \| None Total cost in USD (if the provider reports it)
duration_ms int \| None Wall-clock time for the prompt, in milliseconds

Access these directly on the result without needing a recorder:

result = await call_ai_once("Hello", ai_provider="gemini", ai_model="gemini-2.5-flash")

if result.usage:
    print(f"Tokens: {result.usage.input_tokens} in, {result.usage.output_tokens} out")
    print(f"Cost: ${result.usage.cost_usd or 0:.6f}")
    print(f"Duration: {result.usage.duration_ms}ms")

Note: cost_usd is None when the provider does not report cost data. Always check before using it in calculations.

Advanced Usage

Storing Records in a Database

A practical recorder that appends to a list (replace with your database client):

import asyncio
from pi_sidecar_client import AIResult, call_ai_once, set_usage_recorder

usage_log: list[dict] = []


async def db_recorder(*, request_id, result, call_type, prompt_chars, ai_provider, ai_model):
    record = {
        "request_id": request_id,
        "call_type": call_type,
        "provider": ai_provider,
        "model": ai_model,
        "input_tokens": result.usage.input_tokens if result.usage else 0,
        "output_tokens": result.usage.output_tokens if result.usage else 0,
        "cost_usd": result.usage.cost_usd if result.usage else None,
    }
    usage_log.append(record)


async def main():
    set_usage_recorder(db_recorder)

    result = await call_ai_once(
        "What is the speed of light?",
        ai_provider="gemini",
        ai_model="gemini-2.5-flash",
    )
    await result.record_usage(
        request_id="req-001",
        call_type="question",
        ai_provider="gemini",
        ai_model="gemini-2.5-flash",
    )

    print(f"Usage log: {len(usage_log)} entries")
    for entry in usage_log:
        print(f"  {entry}")


asyncio.run(main())

Using a Synchronous Callback

Your recorder doesn't have to be async. A plain def works too:

def simple_recorder(*, request_id, result, call_type, prompt_chars, ai_provider, ai_model):
    print(f"[{call_type}] {result.usage.input_tokens if result.usage else 0} tokens in")

set_usage_recorder(simple_recorder)

The client detects whether the callback is a coroutine and handles both cases automatically.

Tracking Usage Across Parallel Calls

When running concurrent AI calls with run_parallel_with_limit, record usage inside each task. See Running Parallel AI Calls for concurrency patterns.

from pi_sidecar_client import call_ai_once, run_parallel_with_limit, set_usage_recorder

set_usage_recorder(db_recorder)  # Register once


async def analyze(item: str, idx: int):
    result = await call_ai_once(
        f"In one sentence, what is {item}?",
        ai_provider="gemini",
        ai_model="gemini-2.5-flash",
    )
    await result.record_usage(
        request_id=f"batch-{idx}",
        call_type="analyze",
        ai_provider="gemini",
        ai_model="gemini-2.5-flash",
    )
    return result


items = ["Python", "Kubernetes", "WebAssembly"]
results = await run_parallel_with_limit(
    [analyze(item, i) for i, item in enumerate(items)],
    max_concurrency=3,
)

Tracking Multi-Turn Sessions

For multi-turn conversations using call_ai, record usage after each exchange. See Managing Session Lifecycle for session reuse details.

from pi_sidecar_client import call_ai, get_sidecar_client

result1 = await call_ai("What is Python?", ai_provider="gemini", ai_model="gemini-2.5-flash")
await result1.record_usage(request_id="turn-1", call_type="question", ai_provider="gemini", ai_model="gemini-2.5-flash")

result2 = await call_ai("How does its GIL work?", session_id=result1.session_id, ai_provider="gemini", ai_model="gemini-2.5-flash")
await result2.record_usage(request_id="turn-2", call_type="followup", ai_provider="gemini", ai_model="gemini-2.5-flash")

# Clean up the session when done
await get_sidecar_client().delete_session(result2.session_id)

Cost Aggregation

Sum up costs across multiple calls:

total_cost = sum(r["cost_usd"] or 0 for r in usage_log)
total_tokens = sum(r["input_tokens"] + r["output_tokens"] for r in usage_log)
print(f"Total: {total_tokens} tokens, ${total_cost:.4f}")

Troubleshooting

result.usage is None This happens when the AI call fails before the model returns any data. Check result.success and result.error first — usage data is only available for calls that reached the model.

cost_usd is None even on successful calls Not all providers report cost data. The field is None when the provider doesn't include cost information in its response. Token counts (input_tokens, output_tokens) are still available.

Callback exceptions are silently swallowed By design, record_usage() never raises — if your callback throws an exception, it is caught and logged at debug level. This prevents usage tracking failures from breaking your main workflow. Set PI_SIDECAR_LOG_LEVEL=DEBUG to see these errors:

export PI_SIDECAR_LOG_LEVEL=DEBUG

No data is being recorded Verify both steps are in place: set_usage_recorder(callback) must be called before result.record_usage(). If no recorder is registered, record_usage() silently returns without doing anything.
